# n8n Automation Plan

Future n8n workflows should create GitHub pull requests or commits containing draft MDX files. They should not publish posts automatically.

Recommended flow:
1. Trigger from schedule, webhook, GitHub event, or manual run.
2. Collect source data from commits, logs, raw notes, terminal output, or study notes.
3. Sanitize private data before summarization.
4. Summarize and classify with prompts in `automation/prompts`.
5. Create a Markdown or MDX draft in the appropriate collection under `apps/web/src/content`.
6. Commit to GitHub with `draft: true`, `autoGenerated: true`, `reviewed: false`, and `reviewStatus: "needs-review"`.
7. Let GitHub Pages deploy only after a human review changes the draft status. Optional Raspberry Pi deployment remains a separate documented path.

Do not store API keys in workflow JSON. Use n8n credentials or environment variables.
