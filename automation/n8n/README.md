# n8n Automation Plan

Future n8n workflows should create GitHub pull requests or commits containing draft MDX files.

Recommended flow:
1. Trigger from schedule, webhook, GitHub event, or manual run.
2. Collect source data from commits, logs, notes, or issue updates.
3. Summarize with an AI model using prompts in `automation/prompts`.
4. Create a Markdown or MDX draft in `apps/web/src/content/auto-archive`.
5. Commit to GitHub with `draft: true` and `autoGenerated: true`.
6. Let the Raspberry Pi deploy on the next scheduled pull or manual deploy.

Do not store API keys in workflow JSON. Use n8n credentials or environment variables.
