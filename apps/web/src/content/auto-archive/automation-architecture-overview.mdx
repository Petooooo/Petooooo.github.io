## Publication Model

Generated content is useful only if the workflow preserves trust. This archive uses a draft-first model:

1. Collect source data.
2. Summarize and classify.
3. Generate an MDX draft.
4. Commit to GitHub.
5. Review in Decap CMS or an editor.
6. Publish by setting `draft: false`.

## Review Checklist

- Confirm the source material is real.
- Remove private data.
- Check commands and dates.
- Add context that automation could not infer.
- Publish only when the note is worth keeping.

## What Should Stay Unpublished

- Routine activity summaries with no decision or lesson.
- Logs that expose private systems.
- Drafts where the source cannot be checked.
- Confident explanations for changes that were not actually inspected.
