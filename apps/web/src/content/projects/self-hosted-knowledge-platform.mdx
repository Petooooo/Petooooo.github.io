## Purpose

This repository is the place where site code, technical writing, deployment notes, and automation experiments can live together without depending on a hidden CMS or database.

The first public target is GitHub Pages because it is simple and easy to verify. The longer-term target is still Raspberry Pi hosting with Docker Compose and Caddy, but that path should stay documented instead of rushed.

## Architecture

- Astro builds the static site.
- Content lives in typed MDX collections.
- GitHub remains the source of truth for code and writing.
- Decap CMS is wired as a manual editor over the same repository.
- Docker Compose and Caddy are kept for the Raspberry Pi deployment path.
- GitHub Pages is preserved as the current public deployment target.

## Current Tradeoffs

There is no database, account system, or server-rendered runtime. That is intentional for now: the site should be easy to build, inspect, back up, and redeploy on small hardware.

The main risk is content quality. A static archive only feels useful if the notes are specific, reviewed, and honest about what is still unfinished.
